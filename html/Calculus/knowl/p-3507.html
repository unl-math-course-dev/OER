The function is being evaluated at \(x_1 = 1.4\text{,}\) \(x_2 = 1.8\text{,}\) \(x_3 = 2.2\text{,}\) \(x_4 = 2.6\text{,}\) and \(x_5 = 3.0\text{.}\) Thus, since we know this is a right Riemann sum, we are using \(5\) subintervals with \(\triangle x = 0.4\text{,}\) and \(x_0 = 1.0\text{.}\) From the pattern, we observe that the function must be \(f(x) = x^2 + 1\) on the interval \([1,3]\text{.}\)

  2. If \(S\) is a left Riemann sum instead, then the \(x_i\)'s shift left: \(x_0 = 1.4\text{,}\) \(x_1 = 1.8\text{,}\) \(\ldots\text{,}\) \(x_5 = 3.4\text{.}\) In this setting, the function is identical, but we are considering it on the interval \([1.4, 3.4]\text{.}\)

    If \(S\) is a middle Riemann sum, then (for example) \(1.4\) is the midpoint of an interval \(\triangle x = 0.4\) units long, so \(x_0 = 1.2\) and \(x_1 = 1.6\text{.}\) Continuing that trend, we'd be considering the function \(f(x) = x^2 + 1\) on the interval \([1.2, 3.2]\text{.}\)

  3. \(S\) is an approximation of the area that lies under \(f(x) = x^2 + 1\) on \([1,3]\text{.}\)

  4. To use twice as many subintervals, we move to \(\triangle x = 0.2\) and thus \(x_1 = 1.2\text{,}\) \(x_2 = 1.4\text{,}\) and so on, with the general right endpoint being given by \(x_i = 1 + 0.2 \cdot i\text{.}\) We'll also have \(n = 10\text{,}\) so

    \begin{equation*} R_{10} = \sum_{i=1}^{10} \left( (1+0.2i)^2 + 1 \right) \cdot 0.2\text{.} \end{equation*}
